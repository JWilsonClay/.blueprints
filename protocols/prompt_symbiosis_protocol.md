# Operational Protocol: Prompt-Codebase Symbiosis (OP-SYMBIO-PCSF)

**Purpose:** To govern the recursive relationship between meta-layer blueprints and implementation-layer code.

## 1. Pillars of Symbiosis

- **Modularity:** Every component MUST live in its own hardened artifact for versioning.
- **Constraint-Driven Precision:** Negative constraints and schemas MUST eliminate LLM ambiguity.
- **Self-Referential Evolution:** Artifacts MUST reference and improve each other iteratively.

## 2. Proposal Construction Logic

1.  **Map Element to Artifact:** Every element in the repository structure MUST map to an explicit implementation file.
2.  **Identify Structural Overlap:** Trace shared formats (ASCII, Logging, Tables).
3.  **Trace Dependency:** Define which file enables or constrains another.
4.  **Label Importance:** Apply CRITICAL, HIGH, MEDIUM, LOW hierarchy.
5.  **Apply Safety Metrics:** Zero tolerance for contradictions; no hallucinated functionality.

## 3. The Repository Meta-Pattern

The agent MUST organize prompts into a modular repository structure using the following components:
- `ROLE_OR_PERSONA.md`: Identity assignment.
- `CONTEXT.md`: Environmental grounding.
- `CONSTRAINTS.md`: Logical boundaries.
- `SUCCESS_METRICS.md`: Evaluation criteria.
- `EXAMPLES.md`: Few-shot demonstrations.
- `OUTPUT_FORMAT.md`: Structure specifications.
- `REASONING_GUIDANCE.md`: Chain-of-thought logic.
- `EDGE_CASES.md`: Anomaly handling.

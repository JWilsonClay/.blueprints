# Operational Protocol: Substrate Core Components (OP-SUBSTRATE-COMP)

**Protocol ID:** OP-SUBSTRATE-COMP
**Assigned Role:** Orchestrator_Agent
**Target Audience:** Documentation_Engineer, Verification_Agent
**Purpose:** To enforce a standardized, 12-checklist schema for substrate auditing, ensuring exhaustive mapping of all mechanical and logical components.

## 1. Compliance Standards

- This protocol mandates the generation of 12 distinct checklist files.
- Each checklist MUST follow the exact tabular schema defined in Section 2.
- **Ventilated Prose Enforcement**: This protocol inherits strict lineation rules.
- Rigid lineation is inherited from `OP-RISK-AUDIT`.
- Every row in a generated checklist table MUST be terminated by a literal newline (`\n`).

## 2. Checklist Inventory & Schemas

### 2.1 Standard Table Schema
| [Name]       | [Description]        | [File Name]     | [Line Numbers] | [Logging Integration] |
|---           |---                   |---              |---             |---                    |
| (Identifier) | (Behavioral Context) | (Relative Path) | (Start-End)    | (Log Level/Tool)      |

### 2.2 Mandatory Checklists
1.  **FEATURES_CHECKLIST.md**: High-level business logic and value drivers.
2.  **FUNCTIONS_CHECKLIST.md**: Reusable procedural blocks.
3.  **CLASSES_CHECKLIST.md**: Object-oriented blueprints and state containers.
4.  **MODULES_CHECKLIST.md**: Organizational units and package namespaces.
5.  **VARIABLES_CHECKLIST.md**: Global/Config-level state definitions.
6.  **DATA_STRUCTURES_CHECKLIST.md**: Collections and memory-efficient containers.
7.  **ALGORITHMS_CHECKLIST.md**: Procedural logic for complex problem solving.
8.  **APIS_CHECKLIST.md**: External and internal communication interfaces.
9.  **TESTS_CHECKLIST.md**: Correctness and validation suites.
10. **CONFIGURATIONS_CHECKLIST.md**: Runtime settings and environment payloads.
11. **DEPENDENCIES_CHECKLIST.md**: Third-party libraries and core toolkits.
12. **DOCUMENTATION_CHECKLIST.md**: Inline and strategic written artifacts.

## 3. Negative Constraints

- **Do Not Hallucinate**: If logging is missing, write "None".
- **Do Not Skip Trivialities**: Every function/variable/class must be mapped.
- **Do Not Consolidate**: Each logical entity must occupy exactly one physical line.
- **Do Not Redefine Formatting**: Use `table_formatter.py` to align all checklists post-generation.

## 4. Execution Workflow

- **Input**: Source substrate manifest generated by `.blueprints/search_copy.py`.
- **Action**: Map manifest elements to the relevant `*_CHECKLIST.md` file.
- **Output**: 12 markdown files saved to the `docs/checklists/` directory.
